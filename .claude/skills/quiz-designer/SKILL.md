# quiz-designer（测一测内容设计）

## 何时使用

- 需要从 0 设计一套「测一测」：**维度 / 题目 / 计分 / 结果页 / 传播** 任一环。
- 用户给出主题或参考案例（如 SBTI），需输出 **Markdown / JSON / full 文案** 规格。
- 需区分结果形态并选对方法论（见下表，含「人格模式匹配」等扩展）。

## Instructions

1. **定类型**：先按「结果形式」锁定下表中的类型之一，再展开维度与题量；勿混用计分逻辑。
2. **定维度**：2–8 个维度；定极性、类型数量（4/8/16 等）与是否需要彩蛋/隐藏型（约 10%–20% 触发概率）。
3. **写题目**：题干口语化、情境化；选项 2–4 个、有区分度；控制脏度；必要时加验证题防瞎选。
4. **写计分**：说明维度映射、权重、归一化、平局与兜底规则。
5. **写结果页**：标签 + 短描述 + 行动钩子；给出分享语与话题标签建议。
6. **写传播**：分享触发点、评论区预埋、二创/海报等节点。

## Quiz 类型谱系

根据**结果形式**分为下表类型（含扩展），设计时先确定类型再选对应方法论：

| 类型 | 结果形式 | 典型案例 | 计分逻辑 |
|------|----------|----------|----------|
| **人格类型型** | 离散标签（如 INTJ / 创作者） | MBTI、十六型 | 排除法：题项对应类型，排除其他锁定结果 |
| **光谱连续型** | 连续分值或分档（如每维 L/M/H） | 情感依赖度等 | 维度叠加 + 归一化或分档，再映射区间 / 模式 |
| **人格模式匹配型**（典型扩展） | 离散标签 + 多维分档向量与库内模式比对 | **SBTI**（25 常规 + 兜底 + 彩蛋） | 每维得分 → L/M/H → 与预设 15 位「标准模式」算距离与相似度，排名取最匹配 |
| **知识答题型** | 正确率/分数 | 百科挑战 | 答对计分，累加或加权求和 |
| **数值测算型** | 具体数值（如「买房要 X 年」） | 财富自由计算器 | 公式计算，结果由输入变量决定 |

## 核心能力要点

### 心理底层（简）

- **三大缺口**：自我确认（我是谁）、社交货币（可比较的标签）、确定性安慰（给量化结论）。缺一则传播力明显下降。
- **售卖本质**：用户转发的是「被看见」的感觉；结果要同时服务标签、共鸣与可讨论性。
- **脏度**：题目可生活化甚至带梗以增强参与；敏感题用安全选项或降权，避免失控争议。

### 类型光谱设计

- 测评维度 2–8 个；设定维度极性（正向/负向或两端对立）。
- 类型数量 4/8/16 等，过多传播衰减；可设计彩蛋/隐藏类型。

### 题目设计

- 题干：口语化、情境化，避免说教。
- 选项：2–4 项；有区分度；敏感题设安全选项或降权。
- 引导：开头暖场、中间过渡、结尾可轻暗示结果。
- 防瞎选：同维度两道题措辞不同作一致性校验。

### 评分机制

- **维度计分**：每题映射到 1–2 个维度，加权求和。
- **归一化**：各维度原始分 → 0–100 标准分（如需要）。
- **归类**：类型型用最高维/排除法；光谱型用分值区间。
- **平局**：并列时优先推荐积极结果（若产品无特殊规则）。

### 结果页与传播

- 结果页：标题（类型名 + 情绪 hook）、2–3 句描述、关键词标签、行动钩子。
- 传播：分享触发、话题植入、评论区运营、海报/九宫格/话题标签。

## 调用约定（斜杠命令）

```
/quiz-designer <主题> [选项]
```

| 选项 | 说明 |
|------|------|
| `--type` | 结果类型：`personality` / `spectrum` / `knowledge` / `calc` |
| `--dimensions` | 维度数量，默认 4 |
| `--cases` | 参考案例，如 `SBTI` |
| `--output` | 输出格式：`markdown` / `json` / `full`（含完整文案） |

**示例：**

```
/quiz-designer "职场性格测试"
/quiz-designer "恋爱人格测试" --type personality --dimensions 4
/quiz-designer "你在北京实现财务自由要多久" --type calc
/quiz-designer "创意能力测评" --output full
```

## 默认交付物

1. **类型光谱图**（维度与类型定义）
2. **题目清单**（题干 + 选项 + 计分规则）
3. **评分算法说明**（维度权重、归类逻辑）
4. **结果页文案模板**（标题 + 描述 + 分享语）
5. **传播节点规划**（发布时机、话题标签、互动设计）

## 案例：SBTI（人格模式匹配型）

**SBTI 测一测**是网络流传的趣味人格测试（名称与 MBTI 风格相近，但**并非**四字母 E/I、S/N、T/F、J/P 投票拼型）。官方实现为：**15 个子维度**，每维 2 题 × 3 选项（1/2/3 分）→ 汇总为 **L / M / H** 三档，得到 15 位等级序列，再与库内 **25 种常规人格**的预设「标准模式」比对相似度；另有 **兜底人格**与**隐藏彩蛋人格**。设计测一测时可将其归为「多维分档 + 向量与标准答案距离」一类。

### 时间线与参考入口（便于溯源）

- 2025-04-09 起在 B 站等内容侧传播；测试入口示例：<https://sbti.unun.dev/>、<https://fancc.de5.net/projects/sbti/>；视频：<https://www.bilibili.com/video/BV1LpDHByET6/>。
- 同期曾出现仿站/夺舍站点（如 `sbti.dev` 等），运营时需注意品牌与数据归属表述。

### 十五维结构（五大块 × 每块 3 维）

| 模型 | 维度代号 | 含义 |
|------|----------|------|
| 自我模型 S | S1 / S2 / S3 | 自尊自信、自我清晰度、核心价值 |
| 情感模型 E | E1 / E2 / E3 | 依恋安全感、情感投入度、边界与依赖 |
| 态度模型 A | A1 / A2 / A3 | 世界观倾向、规则与灵活度、人生意义感 |
| 行动驱力 Ac | Ac1 / Ac2 / Ac3 | 动机导向、决策风格、执行模式 |
| 社交模型 So | So1 / So2 / So3 | 社交主动性、人际边界感、表达与真实度 |

### 计分与归类（与官方一致）

1. **答题**：每维 2 道题，每题选 A/B/C 对应 **1 / 2 / 3** 分；该维 raw 分 ∈ [2, 6]。
2. **分档**：2–3 → **L**，4 → **M**，5–6 → **H**。15 维依次得到一串 15 个字母，如 `H H L - H M H - …`（展示时可加空格或连写）。
3. **模式比对**：将 L/M/H 视为 **1 / 2 / 3**，与用户向量与每种人格的 **15 位标准模式**逐位求差的绝对值并求和为「总距离」。最大理论距离为 15×2=**30**。
4. **相似度**：`相似度 = (1 − 总距离/30) × 100%`。
5. **选人**：与 25 种常规人格逐一算距离，**距离小优先**；距离相同时 **完全命中位数多**优先；再相同则 **相似度高**优先。第一名即主结果人格。
6. **兜底**：若第一名相似度**低于 60%**，结果为 **HHHH（傻乐者）**。
7. **彩蛋**：若在官方定义的「特殊题」链路中触发指定选项，**跳过上述匹配**，直接为 **DRUNK（酒鬼）**（详见 `.claude/commands/sbti.md`）。

### 特殊人格

| 代号 | 名称 | 条件 |
|------|------|------|
| HHHH | 傻乐者（兜底） | 与所有常规人格的最高相似度仍低于 60% |
| DRUNK | 酒鬼（隐藏） | 特殊题链路中选定「将白酒灌在保温杯当白开水喝」等官方设定选项 |

### 状态机（对话内执行）

```
INIT → QUESTIONS → RESULT → DONE
```

- `INIT`：简短说明 SBTI 规则与题量（30 道常规题 + 官方特殊题），`q=1`。
- `QUESTIONS`：按官方顺序展示题目与 **A/B/C** 选项，记录 1/2/3 分；遇特殊题分支按 `sbti.md` 处理。
- `RESULT`：生成分档向量 → 排名 / 兜底 / 彩蛋判定 → 输出对应人格的**开场白 + 描述**（可从官方文案摘取）。
- `DONE`：引导分享、重测；可附参考链接。

### 交互与输入

- 用户发起：`/sbti` 或等价入口。
- 接受：`A` / `B` / `C`、`(A)`、中文「选 A」等，不区分大小写（与官方三选项一致）。

### 传播钩子（示例）

- 分享语：`我测出来是 THIN-K，你也来试试？`
- 话题：`#SBTI #测一测`
- 评论预埋：`晒出你的人格代号`

### 题库结构规范（对齐 SBTI）

- 每维 **2** 道题；每题 **3** 选项，分值 **1、2、3**；题干与选项文案见 `.claude/commands/sbti.md` 全表。
- 设计同类测验时：先锁定维度定义，再保证每维题量对称，最后为每种结果维护一条「标准模式」向量并写清距离与兜底规则。
