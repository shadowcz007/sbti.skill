# sbti.skill

面向 **Claude Code / Claude** 的测一测内容设计技能与 **SBTI** 性格测试斜杠命令集合。仓库本身不包含可独立运行的 Web 或 CLI 应用，而是通过 `.claude/` 下的技能与命令定义，让对话里的助手按统一规格设计题库、计分与结果页，并执行 SBTI 测试流程。

## 仓库结构


| 路径                                      | 说明                                                   |
| --------------------------------------- | ---------------------------------------------------- |
| `.claude/skills/quiz-designer/SKILL.md` | **quiz-designer** 技能：从主题拆解测一测规格（题型谱系、维度与计分、结果页与传播节点） |
| `.claude/commands/sbti.md`              | **sbti** 斜杠命令：运行 SBTI 测试的状态机与交互约定                    |


将本仓库（或上述目录）置于 Claude Code 可识别的项目配置中后，即可在对应环境里使用这些技能与命令。

## quiz-designer（测一测内容设计）

**适用场景**：从零设计「测一测」——维度、题目、计分、结果页、传播任一环；或需要 Markdown / JSON / 完整文案形式的交付物。

技能将结果形式分为四类，并分别给出方法论：

- **人格类型型**：离散标签（如十六型）
- **光谱连续型**：连续分值与区间
- **人格模式匹配型**：多维 L/M/H 等与标准模式比对（**SBTI** 见技能内案例）
- **知识答题型**：正确率 / 分数
- **数值测算型**：公式与输入变量得出具体数值

默认交付物包括：类型与维度定义、题目清单、评分说明、结果页模板、传播节点规划。

在支持斜杠命令的环境中可约定为：

```text
/quiz-designer <主题> [--type ...] [--dimensions N] [--cases SBTI] [--output markdown|json|full]
```

详见 `.claude/skills/quiz-designer/SKILL.md` 中的选项表与示例。

## sbti（SBTI 测一测）

**SBTI** 为网络流传的趣味人格测试：**15 个子维度**（五大心理模型各 3 维），每维 2 题 × 三选项计分 → 汇总为 **L/M/H**，再与 **25 种常规人格**的 15 位「标准模式」比对距离与相似度；相似度不足时有 **HHHH（傻乐者）** 兜底，特殊题可触发隐藏 **DRUNK（酒鬼）**。名称易与 MBTI 混淆，但**并非**四字母 E/I、S/N、T/F、J/P 拼型。

使用方式：

```text
/sbti [用户信息]
```

状态流转：`INIT → QUESTIONS → RESULT → DONE`。常规题为 **q1–q30**（选项 A/B/C），其后为官方 **特殊题** 门控链。交互上可识别 `A`/`B`/`C` 及常见中文选答。

完整题库、计分公式、27 种人格文案与参考链接见 `.claude/commands/sbti.md`。

## 与 quiz-designer 的配合

命令文档建议：在出题前调用与 SBTI 相关的 quiz-designer 方案（例如 `SBTI --type spectrum --cases SBTI`），再结合 `/sbti` 传入的用户信息做情境适配，然后按同一状态机执行测试。
